# Authentication 
- security measure that ensures individuals or entities are who they claim to be during a communication or transaction 
### (Knowledge Factor) Something we Know 
- relies on info that a user can recall 
### (Possession Factor) We have 
- relies on the user presenting a physical item to authenticate themselves 
### (Inherence Factor) We are
- relies on the user providing a unique physical or behavioral characteristic of the person to validate they are who they claim to be 
### (Action Factor) What we do 
- relies on the user conducting a unique action to prove who they are 
### (Location Factor) Where we are 
- relies on the user being in a certain geographic location before access is granted 
# Multifactor Authentication (MFA)
- 2 or more authentication methods 
- security process that requires users to provide multiple methods of ID to verify their ID 
# 2-Factor Authentication (2FA)
- 2 authentication methods

# Authorization 
- permissions & privileges granted to users or entities after they've been authenticated 
- users with access to a system determine the actions they can perform, the data they can view or modify, the areas they can access 
- auth mechanisms range from role-based to rule-based to attribute-based controls to determine permissions 
- set of rules & policies that's used to dictate what actions users can perform once verified 
- serves as gatekeeper ensuring correct access to appropriate users 
### 1 Protect Sensitive Data 
### 2 Maintain System Integrity in Organizations 
### 3 Create more Streamlined User Experience

# Accounting 
- security measure that ensures all user activities are properly tracked & recorded 
	- Transparency
	- Security 
	- Accountability 
- what accounting tracks v
	- logging into the system 
	- accessing files 
	- modifying configuration settings 
	- downloading or installing software 
	- attempting unauthorized actions on systems & networks 
### Audit Trail 
- provides chronological record of all user activities that can be used to trace changes, unauthorized access, or anomalies back to a specific user or point in time 
### Regulatory Compliance 
- maintains a comprehensive record of all users' activities 
### Forensic Analysis 
- uses detailed accounting & event logs that can help cybersec experts understand what happened, how it happened, and how to prevent similar incidents from occurring again in the future 
### Resource Optimization 
- orgs can optimize performance & minimize costs by tracking resource utilization and allocation decisions 
### User Accountability 
- thorough accounting system ensures users' actions are monitored and logged, deterring potential misuse & prompting adherence to the orgs policies 
# Accounting Tools
### Syslog Servers
- used to aggregate logs from various network devices & systems so that system admins can analyze them to detect patterns or anomalies in the orgs systems 
### Network Analyzers 
- used to capture & analyze network traffic to gain detailed insights into all the data moving within a network 
### (SIEM) Security Info & Event Management 
- provides real-time analysis of security alerts generated by various hardware & software infrastructure in an org
